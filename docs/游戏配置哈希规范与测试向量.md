# 游戏配置哈希规范与测试向量

## 1. 适用范围

本文定义游戏配置 `configHash` 的统一计算方式，供以下组件共同使用：

- OTA Android 应用；
- Unity 游戏应用；
- 后端配置下发与验收程序。

本文只定义游戏配置 JSON 的规范化与 SHA-256 计算，不改变现有游戏 APK 升级、球盘固件升级和 OTA 自升级流程。

## 2. 固定参数

```text
规范版本：1
字符编码：UTF-8
摘要算法：SHA-256
摘要格式：64 位小写十六进制
```

## 3. 规范化规则

### 3.1 对象

1. 按对象 key 的原始 Unicode 码点序列升序排列。
2. 对每个 value 递归执行相同规范化规则。
3. 不保留输入 JSON 中的空格、换行或缩进。

### 3.2 数组

1. 数组元素顺序保持不变。
2. 每个数组元素递归执行相同规范化规则。

### 3.3 数字

数字按任意精度十进制处理：

```text
1          -> 1
1.0        -> 1
1e0        -> 1
-0         -> 0
1.2300     -> 1.23
0.00000100 -> 0.000001
```

要求：

- 禁止使用科学计数法输出；
- 去除小数尾随零；
- 去除无意义的小数点；
- 正零和负零统一输出为 `0`；
- 不允许 `NaN`、`Infinity` 或 `-Infinity`。

### 3.4 字符串

1. 使用双引号包裹。
2. 双引号转义为 `\"`。
3. 反斜杠转义为 `\\`。
4. 换行、回车和制表符按 JSON 标准转义。
5. 其他控制字符使用小写 `\u00xx`。
6. `/` 不转义。
7. 中文、emoji 等非控制 Unicode 字符直接以 UTF-8 编码，不转换成 `\uXXXX`。
8. 不执行 HTML 字符转义。

### 3.5 其他 JSON 值

```text
true  -> true
false -> false
null  -> null
```

### 3.6 最终哈希

1. 得到无空格、无换行、无缩进的规范化 JSON 字符串。
2. 将字符串编码为 UTF-8 字节。
3. 计算 SHA-256。
4. 输出 64 位小写十六进制字符串。

## 4. 标准测试向量

机器可读版本位于：

```text
app/src/test/resources/game_config_hash_vectors.json
```

### V01 空对象

输入及规范化结果：

```json
{}
```

SHA-256：

```text
44136fa355b3678a1146ad16f7e8649e94fb4fc21fe77e8310c060f61caaff8a
```

### V02 对象 key 排序

输入：

```json
{"b":2,"a":1}
```

规范化：

```json
{"a":1,"b":2}
```

SHA-256：

```text
43258cff783fe7036d8a43033f830adfc60ec037382473548ac742b888292777
```

### V03 数字规范化

输入：

```json
{"e":0.00000100,"d":1.2300,"c":-0,"b":1e0,"a":1.0}
```

规范化：

```json
{"a":1,"b":1,"c":0,"d":1.23,"e":0.000001}
```

SHA-256：

```text
b5f8b73ca6050d4f95dfc7c0cab0034401261955b4a11945ccbd133f7dccd0c3
```

### V04 嵌套对象和数组

输入：

```json
{"z":[3,{"b":2,"a":1},1],"a":{"y":false,"x":null}}
```

规范化：

```json
{"a":{"x":null,"y":false},"z":[3,{"a":1,"b":2},1]}
```

SHA-256：

```text
0e4b68196889e9c5b45eedc202f906599cec476cba7a9405d09943da5ab2d801
```

### V05 Unicode 和字符串转义

输入：

```json
{"提示":"配置成功","emoji":"弹珠🎮","text":"引号\"和反斜杠\\以及换行\n"}
```

规范化：

```json
{"emoji":"弹珠🎮","text":"引号\"和反斜杠\\以及换行\n","提示":"配置成功"}
```

SHA-256：

```text
39333ebd8b5cddb6ec9727b063bd99c3c3d128d26ccf328ae254ebbee2eb6840
```

其中 `\n` 是 JSON 转义序列，规范化字符串中不得写入真实换行。

输入：

```json
{"sound":{"effectVolumePercent":40,"musicVolumePercent":60},"difficulty":{"mode":"EASY"},"lighting":{"stripBrightnessPercent":80,"panelBrightnessPercent":70,"playfieldBrightnessPercent":90}}
```

```json
{"difficulty":{"mode":"EASY"},"lighting":{"panelBrightnessPercent":70,"playfieldBrightnessPercent":90,"stripBrightnessPercent":80},"sound":{"effectVolumePercent":40,"musicVolumePercent":60}}
```

SHA-256：

```text
329efc9e24be1f153f99528c9d70e1461c3c702e7ba72c144daed56de71d36fd
```

```json
{"difficulty":{"smartModeSettings":{"profitCycleDays":7,"cardExchangeAmountCent":500,"profitRatePercent":35},"mode":"SMART"},"lighting":{"playfieldBrightnessPercent":100,"stripBrightnessPercent":85,"panelBrightnessPercent":75},"sound":{"musicVolumePercent":50,"effectVolumePercent":65}}
```

```json
{"difficulty":{"mode":"SMART","smartModeSettings":{"cardExchangeAmountCent":500,"profitCycleDays":7,"profitRatePercent":35}},"lighting":{"panelBrightnessPercent":75,"playfieldBrightnessPercent":100,"stripBrightnessPercent":85},"sound":{"effectVolumePercent":65,"musicVolumePercent":50}}
```

```text
3ce79f4afcc1f824616335f957c3d3a41ebd3ff3549e298fce02a311852c0994
```

### V08 数组顺序和数字统一

输入：

```json
{"values":[1.0,1e0,-0,1.2300,{"b":2,"a":1}]}
```

规范化：

```json
{"values":[1,1,0,1.23,{"a":1,"b":2}]}
```

SHA-256：

```text
efac387d4f52d1c4559fe3b1912ed5939fdc5d6d285ae91566125d1582b5706b
```

## 5. 验收要求

OTA、Unity 和后端实现必须同时满足：

1. 8 组测试向量全部得到本文规定的规范化字符串。
2. 8 组 SHA-256 全部完全一致。
3. 输入对象 key 顺序变化不得改变最终哈希。
4. 数组元素顺序变化必须改变最终哈希。
5. `1`、`1.0` 和 `1e0` 必须得到相同的规范化数字和哈希结果。
6. 哈希计算必须针对配置对象 `config` 本身，不包含 `messageId`、`configVersion`、`configHash` 等外层字段。
